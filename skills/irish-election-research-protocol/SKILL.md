# Irish Election Research Protocol

## 개요
아일랜드 선거 정보를 조사할 때 출처 신뢰도 기준, 검색 순서, 교차검증 규칙, 기록 형식을 표준화한다.
사용자 검토용 리서치와 Codex 자체 사실검증에 동일한 절차를 적용한다.

## 1. 조사 시작 규칙
1. 조사 질문을 한 문장으로 고정한다.
2. 시간 기준을 명시한다. 예: `2026-04-04 기준 최신`.
3. 사실 유형을 먼저 분류한다.
- 법/제도: 법령, 공식 가이드, 위원회 문서 우선
- 운영/일정: 정부·선관위 공지 우선
- 통계/결과: 공식 집계기관 우선
4. 상대 날짜 표현(`today`, `recent`)을 절대 날짜로 변환해 기록한다.

## 2. 출처 우선순위
출처는 아래 우선순위로 찾고, 상위 레벨 출처가 있으면 하위 레벨은 보조로만 사용한다.

1. Level A (최우선, 1차 권위 출처)
- 아일랜드 선거관리위원회: `electoralcommission.ie`
- 아일랜드 법령 데이터베이스: `irishstatutebook.ie`
- Oireachtas(의회 공식): `oireachtas.ie`
- 아일랜드 정부 공식 사이트: `gov.ie`
- 선거/국민투표 공식 집계 사이트: `electionsireland.org`

2. Level B (공공기관/보조 1차)
- 지방정부/카운티 카운슬 공식 사이트
- CSO 등 국가 통계기관의 공식 통계 문서

3. Level C (2차 자료)
- 학술기관, 법률 해설, 언론 분석
- Level A/B 부재 시에만 제한적으로 사용

## 3. 필수 검색 프로세스
1. `Level A 도메인 제한 검색`을 먼저 수행한다.
2. 동일 사실에 대해 최소 2개 독립 출처를 확보한다.
3. 법·규칙 관련 내용은 반드시 `법령/공식 규정 문서`로 역추적한다.
4. 최신성이 중요한 항목은 문서의 `게시일/개정일/시행일`을 반드시 수집한다.
5. 출처 간 충돌이 있으면 가장 최근의 Level A 공식 문서를 우선한다.

## 4. 검증 체크리스트
아래 항목을 모두 만족해야 "검증됨"으로 표시한다.

- 출처 URL이 공식 도메인인가
- 문서 날짜(게시/개정/시행)가 확인되는가
- 주장 문장이 원문과 의미상 일치하는가
- 숫자·임계값·절차 순서(예: quota 계산, 이양 조건)가 원문과 정확히 일치하는가
- 최소 1개 보조 출처에서 동일 사실이 재확인되는가

하나라도 실패하면 "검증 미완료"로 표시하고 부족 항목을 명시한다.

## 5. 기록 포맷
조사 결과를 아래 포맷으로 정리한다.

- 질문:
- 기준일:
- 핵심 답변(요약 3문장 이내):
- 확정 사실:
- 불확실/확인 필요:
- 출처 목록:
  - `[레벨] 기관명, 문서명, URL, 게시/개정일, 확인일`
- 검증 판정: `검증됨` 또는 `검증 미완료`

## 6. 인용 규칙
1. 긴 직접 인용 대신 요약을 기본으로 한다.
2. 인용은 사실 검증에 필요한 최소 범위만 사용한다.
3. 각 핵심 주장 문장 뒤에 대응 출처를 바로 붙인다.
4. 추론이 포함되면 `추론`으로 명확히 표시한다.

## 7. 이 프로젝트에 맞춘 적용 지침
1. STV 시뮬레이션 로직(할당표, surplus 이전, 탈락 순서)은 `법령 + 선관위 가이드` 조합으로 확인한다.
2. UI 설명문(유권자 안내)은 법령 원문 표현보다 사용성 중심으로 요약하되, 근거 URL을 항상 남긴다.
3. 코드/문서 변경 전에 참고 출처를 먼저 수집하고, 변경 후에는 근거 목록을 커밋 메시지 또는 변경 노트에 남긴다.
